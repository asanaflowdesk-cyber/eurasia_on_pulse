# v47 — создание пользователей через модалку

Изменения:

- `+ пользователь` теперь открывает отдельную модалку создания пользователя.
- Карточка доступа используется только для редактирования существующего пользователя.
- В карточке доступа убрана кнопка создания/синхронизации Auth-аккаунта.
- В карточке доступа остались: сохранение прав, удаление профиля доступа, генерация/копирование/сброс пароля.
- Создание нового пользователя одним действием вызывает `pulse-admin-create-user` и создаёт/синхронизирует:
  - Supabase Authentication user;
  - `public.pulse_profiles`.
- Вызов Edge Functions переведён на `supabase.functions.invoke`, чтобы корректно передавались session/JWT и служебные заголовки.
- Для ошибки `Invalid credentials` добавлено более понятное сообщение: проверять `SUPABASE_SERVICE_ROLE_KEY` в Edge Function Secrets и соответствие Supabase-проекта.

Важно: для работы создания/сброса пароля Edge Functions должны быть задеплоены, а в Secrets должны быть `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY`.
